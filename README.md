# KOO 대시보드 (구 트렌드 TOP 10, v3.9)

관심 키워드의 시장현황(네이버 데이터랩)과, 검색어로 직접 조회하는 네이버 쇼핑 검색
결과 TOP10을 함께 보여주는 **개인 참고용** 대시보드입니다. **공식 순위 서비스가
아니라, 네이버 데이터를 화면용으로 정리해서 보여주는 개인 참고·연습 프로젝트**라는
점을 꼭 기억해 주세요.

> **이름 안내**: 저장소(repo) 이름과 배포 주소는 만들 때 이름 그대로
> `trend-top10`을 계속 씁니다. v3.6부터 화면에 표시되는 **서비스 이름만
> "KOO 대시보드"로 바뀌었습니다**(제목, 브라우저 탭 제목, 화면 설명 등). 네이버
> 데이터랩·쇼핑 검색 API를 설명하는 문구는 실제 데이터 출처이므로 그대로
> "네이버"로 남겨뒀습니다 — 서비스 이름과 데이터 출처 설명은 서로 다른 것이라
> 구분했습니다.

v3.0은 새 기능을 추가하는 버전이 아니라, 지금까지 만든 기능을 그대로 유지하면서
화면 문구·화면 흐름·오류 안내·모바일 화면·README를 개인 실사용 기준으로 다시
정리한 **안정화 버전**이었습니다. v3.1은 그 위에서 **기능 추가 없이 디자인만
다시 다듬은 버전**이었고, v3.2는 v3.1 디자인 위에 **상단 헤더에 개인 캐릭터
이모티콘을 얹어 브랜딩 요소를 더한 버전**입니다. v3.3은 **기능 추가가 아니라
v3.2에서 임시로 base64 인라인했던 이모티콘 이미지를 `server.js`의 정적 파일
라우트로 정식 제공하도록 구조만 정리한 버전**입니다. v3.4는 **시장현황
키워드를 화면에서 직접 입력·적용할 수 있게 만든 기능 추가 버전**입니다. v3.5는
**v3.4에서 적용한 시장현황 키워드를 이 브라우저의 localStorage에 저장해서,
다음 접속이나 새로고침 때 자동으로 복원하는 기능 추가 버전**입니다. v3.6은
**기능 추가가 아니라 "KOO 대시보드 A안" 참고 이미지 스타일에 맞춰 전체 화면을
민트·청록 톤으로 다시 디자인한 버전**입니다. v3.7은 **v3.6에서 만든 "핵심
키워드"·"상승세 키워드" 카드에 키워드를 선택해 비교하는 기능을 더한 버전**입니다.
v3.8은 **v3.7의 "선택 키워드 비교" 결과를 CSV 파일로 내려받고, 실제 비교 데이터를
바탕으로 자동 요약 문장을 만들어 복사할 수 있게 한 기능 추가 버전**입니다. v3.9는
**최대 5개까지 등록한 애니메이션의 네이버 검색 관심도를 주간 단위로 비교하는
"애니메이션 트렌드" 화면을 새로 추가한 버전**입니다(1단계에서 서버 API를 먼저
만들고 검증한 뒤, 2단계에서 이 화면을 연결했습니다).

## 이 프로젝트는 무엇인가요? (목적)

"트렌드 TOP 10"은 화면을 아래 순서로 살펴보는 개인 참고용 대시보드입니다.

1. **시장현황**: 미리 정해둔 관심 키워드 10개(선크림, 양산, 여름원피스, 쿨토시,
   제습기, 미니선풍기, 래쉬가드, 샌들, 냉감이불, 물놀이용품)의 최근 30일 검색·쇼핑
   클릭 추이를 네이버 데이터랩(검색어트렌드·쇼핑인사이트) API로 참고합니다.
2. **애니메이션 트렌드**(v3.9): 최대 5개까지 등록한 애니메이션의 네이버 검색
   관심도를 주간 단위로 비교합니다. 시장현황과 마찬가지로 네이버 데이터랩
   API를 쓰지만, 대상은 "등록한 애니메이션끼리의 상대 비교"로 한정됩니다.
3. **상품 검색**: 궁금한 검색어를 직접 입력하면 네이버 쇼핑 검색 API가 실제로
   돌려주는 검색 결과 TOP10(상품명·쇼핑몰·최저가·카테고리 등)을 조회합니다. 기본
   정렬은 정확도순(`sort=sim`)입니다.
4. **상세 결과**: 상품 검색 결과를 조회 기준·결과 요약 문장·가격 요약 카드·카테고리
   필터·순위 표로 자세히 살펴보고, 결과 요약을 텍스트로 복사할 수 있습니다.
5. **사용 안내**: 이 화면을 어떻게 이해하고 써야 하는지, 그리고 운영 점검 링크를
   마지막으로 정리해서 보여줍니다.

이 화면이 보여주는 모든 데이터는 **구매순위·판매순위·네이버 공식 인기순위가
아닙니다.** 빌드 도구나 외부 라이브러리 없이 `index.html` 파일 하나(+ 서버
`server.js`)로 동작합니다. API 연결에 실패하면 예시(샘플) 데이터가 화면에 표시될 수
있습니다.

## 로컬 실행 방법

`index.html`은 화면에 보여줄 데이터를 `server.js`(API 서버)에게 요청해서 받아옵니다.
그래서 아래 순서로 실행해야 정상적으로 데이터가 보입니다.

1. 이 폴더에서 터미널(PowerShell)을 열고 아래 명령어로 서버를 먼저 실행합니다.
   ```
   node server.js
   ```
   `네이버 쇼핑 검색 TOP10 서버 실행 중: http://localhost:3000/...` 메시지가 보이면
   성공입니다.
2. 서버를 켜둔 상태로 `index.html` 파일을 더블클릭해서 브라우저로 엽니다.

서버를 켜지 않고 `index.html`만 열면, "서버 연결 실패" 안내 배너가 화면에 표시되고
직전에 불러온 결과(또는 예시 데이터)가 그대로 남습니다.

로컬에서 실제 네이버 데이터를 보려면 아래 "환경변수 안내"에 따라 `.env` 파일에 API
키를 채워야 합니다. `.env` 파일이 없거나 값이 비어 있으면, 서버는 오류를 던지지 않고
예시 데이터로 자동 대체합니다.

## Render 배포 및 환경변수 안내

### 배포 설정값

[render.com](https://render.com)에서 "New +" → "Web Service"로 새 서비스를 만들 때
아래 값을 그대로 입력하면 됩니다.

| 설정 항목 | 입력할 값 |
| --- | --- |
| Service Type | **Web Service** |
| Language(Runtime) | **Node** |
| Branch | **main** |
| Build Command | `npm install` (또는 비워둬도 됩니다 — 이 프로젝트는 외부 패키지를 쓰지 않습니다) |
| Start Command | `node server.js` |

### 환경변수 채우기

네이버 API는 기능별로 따로 신청하기 때문에, 키가 **한 쌍**만 있을 수도 있고
**두 쌍**(용도별)일 수도 있습니다. 두 경우 모두 지원합니다.

- **키가 한 쌍인 경우** — 아래 두 값만 채우면 두 API(쇼핑인사이트·검색어트렌드) 모두에
  공용으로 사용됩니다.
  ```
  NAVER_CLIENT_ID
  NAVER_CLIENT_SECRET
  ```
- **키가 두 쌍인 경우** — 용도별 값이 있으면 공용 값보다 우선 사용됩니다.
  ```
  NAVER_SHOPPING_CLIENT_ID
  NAVER_SHOPPING_CLIENT_SECRET
  NAVER_SEARCH_TREND_CLIENT_ID
  NAVER_SEARCH_TREND_CLIENT_SECRET
  ```

로컬에서는 이 이름 그대로 `.env` 파일(`.env.example` 참고)에 채우고, Render에서는
"Environment Variables" 설정 화면에 같은 이름으로 등록합니다.

> ⚠️ **중요**: 실제 API 키/Secret 값은 GitHub 저장소(코드)나 `.env` 파일에 커밋해서
> 올리는 것이 아니라, 반드시 Render의 "Environment Variables" 설정 화면에 직접
> 입력해야 합니다. GitHub에는 `.env.example`처럼 값이 비어있는 안내 파일만 올라가야
> 하고, `.gitignore`가 `.env` 파일 자체를 Git에서 제외해줍니다. API 키/Secret은
> `index.html` 같은 브라우저(클라이언트) 코드에는 절대 넣지 않습니다 — 반드시
> `server.js` 쪽 환경변수로만 다룹니다.

### 설정이 잘 됐는지 확인하기

배포 후(또는 로컬에서) `/api/naver-ready` 주소에 접속하면 아래처럼 두 값이 각각
표시됩니다.

```json
{ "shoppingInsightReady": true, "searchTrendReady": true }
```

`false`가 보이면 해당 용도의 환경변수 값(또는 공용 `NAVER_CLIENT_ID`/`SECRET`)을 다시
확인해야 합니다. Render 로그(Logs) 탭에서 서버 실행 메시지가 뜨는지도 함께 확인하면
좋습니다.

## 사용 API 설명

이 대시보드는 네이버 오픈 API 중 아래 3가지를 사용합니다. 모두 `server.js`에서만
호출하며, `index.html`(브라우저)은 이 API들을 직접 호출하지 않습니다.

| API | 사용 위치 | 알려주는 것 |
| --- | --- | --- |
| 데이터랩 검색어트렌드 | `/api/market-status` (1. 시장현황) | 관심 키워드별 검색 관심도 추이 |
| 데이터랩 쇼핑인사이트 | `/api/market-status` (1. 시장현황) | 관심 키워드별 쇼핑 클릭 추이 |
| 데이터랩 검색어트렌드 | `/api/anime-weekly-trends` (2. 애니메이션 트렌드, v3.9) | 등록한 애니메이션끼리 비교한 주간 검색 관심도·순위 |
| 쇼핑 검색 API | `/api/naver-shopping-search` (3/4. 상품 검색·상세 결과) | 검색어 기준 상품 검색 결과 TOP10 |

API 호출이 실패하면(키 미설정, 네이버 서버 오류, 응답에 쓸만한 데이터 없음 등) 각
영역은 독립적으로 예시(샘플) 데이터 또는 직전 결과로 대체되고, 화면에는 그 사실을
알리는 안내 배너가 표시됩니다.

## 검색순위와 가격순위 차이

"3. 상세 결과" 표에는 순위를 나타내는 열이 두 개 있는데, 서로 뜻이 다릅니다.

- **검색순위**: 네이버 쇼핑 검색 API가 **반환한 순서** 그대로입니다(기본 정렬:
  정확도순 `sort=sim`). 진한 파란 숫자로 표시됩니다.
- **가격순위**: **현재 화면에 표시된 결과 안에서** 최저가(`lprice`) 기준으로 낮은
  가격부터 매긴 참고 순위입니다. 파란 배지로 표시되며, 카테고리 필터를 바꾸면 그
  범위 안에서 다시 계산됩니다. 가격 정보가 없는 상품은 `-`로 표시됩니다.

**두 순위 모두 구매순위·판매순위·네이버 공식 인기순위가 아닙니다.** 표 아래 안내
문구와 "4. 사용 안내"에도 같은 설명이 유지됩니다.

## 시장현황의 의미

"1. 시장현황"은 관심 키워드 기준으로 최근 30일 검색 관심도(검색어트렌드)와 쇼핑
클릭 추이(쇼핑인사이트)를 요약해서 보여주는 영역입니다. 기본적으로는
서버(`server.js`)에 미리 정해둔 **관심 키워드 10개**를 쓰지만, v3.4부터는 화면의
"시장현황 키워드" 영역에서 **직접 키워드를 입력해 바꿔볼 수** 있습니다(아래
"시장현황 키워드 직접 입력하기" 참고).

- **네이버 공식 인기검색어 순위가 아니라, 설정한(또는 직접 입력한) 관심 키워드
  기준의 참고용 정보**입니다.
- 관심도 점수는 네이버 데이터랩 API 호출(최대 5개 키워드씩 묶음)마다 각각 0~100으로
  새로 정규화되므로, 서로 다른 묶음에 속한 키워드끼리의 절대 점수 비교는 근사치입니다.
- "변동" 값도 네이버가 제공하는 공식 값이 아니라, 최근 7일 평균과 이전 7일 평균의
  차이를 서버가 직접 계산한 근사치입니다.

### 핵심 키워드 / 상승세 키워드 카드 (v3.6)

v3.6부터 시장현황 요약은 예전의 4칸 카드(관심도 높은 키워드·상승세 키워드·
하락세 키워드·주요 카테고리) 대신 **"핵심 키워드"·"상승세 키워드"** 2장으로
정리했습니다. 두 카드 모두 `/api/market-status` 응답(`keywordData`)에 이미
있는 실제 값만 사용하며, 화면에 새로운 가짜 숫자를 만들어 넣지 않습니다.

- **핵심 키워드 선정 기준**: `keywordData`를 `searchInterestScore`(검색 관심도
  지수, 0~100) 내림차순으로 정렬해 상위 5개를 고릅니다. 기존에 "관심도 높은
  키워드" 카드가 쓰던 기준(`findKeywordWithHighestScore`)을 여러 개로 확장한
  것과 같습니다(`selectCoreMarketStatusKeywords` 함수).
- **핵심 키워드 대표 퍼센트**: 표시된 키워드들의 `searchTrendDiff`(최근 7일 평균 −
  이전 7일 평균) **평균**입니다. "평균 증감(지수)"라고 이름을 그대로 밝혀서
  계산 방식과 라벨이 어긋나지 않게 했습니다.
- **상승세 키워드 선정 기준**: `searchTrend === "up"`이면서 `searchTrendDiff`가
  실제로 양수인 키워드만 증감이 큰 순으로 상위 5개를 고릅니다
  (`selectRisingMarketStatusKeywords` 함수). 상승세로 분류할 키워드가 하나도
  없으면 "현재 기준에서 상승세로 분류할 수 있는 키워드가 없습니다."라는 빈 상태
  문구를 보여줍니다(카드 높이는 그대로 유지).
- **상승세 키워드 대표 퍼센트**: 표시된 키워드들의 `searchTrendDiff` **최댓값**이며,
  라벨도 "최고 상승 폭(지수)"으로 계산 방식과 맞췄습니다.
- **단위 표기(%p)**: `searchTrendDiff`는 실제 매출/판매 증감률이 아니라 네이버
  데이터랩 관심도 지수(0~100) 위에서 계산한 "지수 포인트" 차이이므로, "%"가
  아니라 **"%p"(퍼센트 포인트)**로 표시해서 실제 계산과 다른 인상을 주지
  않습니다. 값을 계산할 수 없으면(표시할 키워드가 없거나 유효한 수치가 없으면)
  "계산 불가"라고 표시하고, 0으로 나누기·`NaN`·`Infinity` 같은 값이 그대로
  노출되지 않도록 안전하게 처리했습니다.
- **미니 그래프**: `/api/market-status` 응답에는 날짜별 시계열(예: 일별 추이
  배열)이 없기 때문에, "시간이 흐르는 그래프"를 가짜로 만들지 않았습니다.
  대신 **지금 화면에 표시된 키워드들의 실제 수치를 나란히 비교하는** SVG
  그래프를 그리고, "키워드 상대 지수 비교"라고 정확한 의미를 캡션에 함께
  표시합니다(핵심 키워드는 `searchInterestScore`, 상승세 키워드는
  `searchTrendDiff` 기준). `Math.random()`은 쓰지 않으므로 같은 데이터면
  항상 같은 그래프가 그려집니다.
- 색상만으로 상승/하락을 표현하지 않도록, 대표 퍼센트에는 항상 ▲(상승)/▼(하락)/
  –(변동 없음) 기호와 텍스트를 함께 표시합니다.

### 선택 키워드 비교 (v3.7)

"핵심 키워드"·"상승세 키워드" 카드의 키워드 칩을 **누르면 그 키워드가 "선택 키워드
비교" 패널(같은 "1. 시장현황" 영역, 두 카드 아래)에 추가**됩니다. 다시 누르면
선택이 풀립니다.

- **선택은 최대 5개**까지 가능합니다. 6번째를 선택하려고 하면 기존 선택은 그대로
  두고 "비교할 키워드는 최대 5개까지 선택할 수 있습니다."라는 안내만 보여줍니다
  (빨간 오류색이 아니라 호박색 안내 톤을 씁니다).
- 같은 키워드가 두 카드에 동시에 있어도 **하나의 선택으로 취급**합니다(칩에
  `data-keyword` 속성을 달아 실제로는 "키워드 이름"을 기준으로 선택 상태를
  관리하고, 화면에 보이는 두 칩 모두 같은 선택 상태로 동기화됩니다).
- 칩은 `<button>` 요소이고 `aria-pressed`로 선택 상태를 나타내므로 Tab으로
  이동해 Enter/Space로도 선택할 수 있습니다.
- 처음 열었을 때는 핵심 키워드 1번째 항목이 기본 선택되어 있습니다(핵심 키워드
  자체가 없으면 아무것도 선택하지 않습니다).
- **비교 패널 구성**: 선택 요약 문구, 정렬 선택(선택 순서 / 관심도 높은 순 /
  상승 폭 높은 순 / 변동 폭 큰 순), 선택 전체 해제 버튼, "검색 관심도 지수"
  막대그래프, "검색 추세 증감 폭"(0 기준 양방향) 막대그래프, 선택 키워드 상세
  표를 보여줍니다. 그래프는 모두 실제 `searchInterestScore`/`searchTrendDiff`
  값으로만 그리며, `Math.random()`이나 가짜 시계열은 쓰지 않습니다. 값이 없거나
  숫자가 아니면(`NaN`/`Infinity` 포함) "계산 불가"로 안전하게 표시합니다.
- 그래프는 장식(`aria-hidden="true"`)으로 처리하고, 같은 데이터를 그래프 아래
  캡션 문장과 아래쪽 표로 한 번 더 제공해서 스크린 리더 사용자도 동일한 정보를
  얻을 수 있게 했습니다.
- **정렬은 화면에 보여주는 순서만 바꿀 뿐, 실제 선택(추가/해제) 순서 자체는
  바뀌지 않습니다.** 동점일 때는 선택 순서 → 키워드명 순으로 안정적으로
  정렬합니다.
- [선택 전체 해제]는 비교 선택만 지웁니다. 시장현황 원본 데이터, 적용된 키워드,
  `trendTop10.marketStatusKeywords`에 저장된 값은 전혀 건드리지 않습니다.
- **비교 선택 상태는 이 브라우저에 영구 저장되지 않습니다.** 새로고침하면
  다시 기본 선택(핵심 키워드 1번째)으로 돌아갑니다 — v3.5의 시장현황 키워드
  자동 복원(`trendTop10.marketStatusKeywords`)과는 별개의, 화면에서만 쓰는
  임시 상태입니다.
- 사용자 키워드 적용/기본값 복원/자동 복원으로 시장현황 데이터가 바뀌면, 새
  데이터에 더 이상 없는 선택 키워드는 자동으로 제거되고, 그 결과 선택이
  0개가 되면 새 핵심 키워드 1번째 항목이 다시 기본 선택됩니다. `/api/market-status`
  요청이 실패하면(네트워크 오류, 502 등) 시장현황 데이터와 비교 선택 상태를
  그대로 두고 아무것도 지우지 않습니다.
- 검색 관심도/증감 폭은 매출·판매량이 아니라는 안내를 비교 패널 하단에 한 곳만
  두었습니다("검색 관심도와 증감 폭은 네이버 데이터 기반 지표이며 실제
  판매량이나 매출을 의미하지 않습니다.").

### 비교 결과 CSV 다운로드·요약 문장 (v3.8)

"선택 키워드 비교" 패널 아래 **"비교 결과 활용"** 영역에서, 지금 선택한 키워드들의
비교 결과를 CSV 파일로 저장하거나 자동 생성된 요약 문장을 복사할 수 있습니다. 선택한
키워드가 없어도 이 영역 자체는 계속 보이고, 버튼만 비활성화됩니다.

- **CSV 다운로드**: [비교 결과 CSV 다운로드]를 누르면 현재 선택된 키워드 데이터를
  CSV 파일로 바로 내려받습니다. 서버를 거치지 않고 브라우저에서 `Blob` +
  `URL.createObjectURL` + 임시 `<a download>` 클릭 + `URL.revokeObjectURL`
  방식으로 만들어지며, 다운로드 후 임시 요소는 DOM에서 바로 제거됩니다.
  - **열 구성(헤더 고정)**: `선택순서,키워드,검색관심도지수,검색추세,검색추세증감폭_퍼센트포인트`
  - **선택 순서 기준**: CSV는 화면의 정렬 드롭다운(선택 순서/관심도 높은 순/상승
    폭 높은 순/변동 폭 큰 순) 상태와 **완전히 무관하게, 항상 키워드를 선택한
    순서 그대로** 담깁니다. 화면의 그래프·표만 정렬 드롭다운을 따르고, CSV와
    아래 요약 문장은 항상 선택 순서만 봅니다.
  - **값이 없는 경우**: 검색 관심도 지수·증감 폭을 계산할 수 없으면(값이 없거나
    숫자가 아니면) 해당 칸을 빈 값으로 남깁니다. `NaN`/`Infinity`/`undefined`/
    `null` 같은 문자열이 그대로 CSV에 노출되지 않도록 안전하게 처리했습니다.
    검색추세 칸은 up→상승/down→하락/flat→보합/그 외→확인 불가로 한글 라벨을
    씁니다. 증감 폭 칸에는 "%p" 기호 없이 숫자만 넣고, 단위는 열 이름
    (`검색추세증감폭_퍼센트포인트`)에서 밝혔습니다.
  - **엑셀 한글 깨짐 방지**: 파일 맨 앞에 UTF-8 BOM을 붙여서 엑셀에서 한글이
    깨지지 않게 했습니다.
  - **CSV 특수문자 이스케이프**: 값에 쉼표·큰따옴표·줄바꿈이 있으면 RFC4180
    규칙대로 전체를 큰따옴표로 감싸고, 내부의 큰따옴표는 두 번 반복해서
    이스케이프합니다(`escapeCsvCell`).
  - **CSV 수식 주입(Formula Injection) 방어**: 사용자가 입력한 키워드 값이
    `=`, `+`, `-`, `@` 또는 탭/캐리지리턴으로 시작하면, 스프레드시트 프로그램이
    이를 수식으로 실행하지 못하도록 앞에 작은따옴표(`'`)를 붙입니다
    (`protectCsvFormula`). 실제 숫자 필드(검색 관심도 지수, 증감 폭)는 음수를
    나타낼 때 정상적으로 `-`로 시작할 수 있으므로 이 방어 대상에서 제외하고
    숫자 그대로 둡니다.
  - **파일명**: `KOO_키워드비교_YYYYMMDD_HHmm.csv` 형식이며, 브라우저의 로컬
    시간 기준입니다(예: `KOO_키워드비교_20260712_0905.csv`).
  - **선택이 없을 때**: 파일을 만들지 않고 "CSV로 저장할 키워드를 먼저 선택해
    주세요."라는 안내만 보여줍니다. 이때는 버튼도 비활성화되어 있습니다.
- **요약 문장 자동 생성**: 키워드를 선택/해제하거나, [선택 전체 해제]를 누르거나,
  시장현황 데이터가 정상적으로 갱신될 때마다(키워드 적용 성공/기본값 복원/저장된
  키워드 자동 복원 모두 포함) 요약 문장이 실제 비교 데이터를 바탕으로 새로
  만들어집니다. **정렬 드롭다운을 바꾸거나 `/api/market-status` 요청이
  실패했을 때는 요약 문장을 다시 만들지 않습니다** — 화면 정렬은 표시 순서만
  바꾸는 것이라 요약과 무관하고, API 실패 시에는 직전의 정상 데이터와 요약을
  그대로 둡니다.
  - **생성 방식**: 외부 AI 호출 없이, 순수 자바스크립트 규칙 기반 템플릿으로
    문장을 조립합니다(`generateComparisonSummaryText`). 새로운 숫자를 만들지
    않고, `/api/market-status` 응답에 이미 있는 `searchInterestScore`/
    `searchTrend`/`searchTrendDiff` 값만 설명·비교합니다.
  - **여러 키워드를 선택한 경우** 포함되는 내용: 선택한 키워드 개수, 검색
    관심도가 가장 높은 키워드와 그 지수, 상승 폭이 가장 큰 키워드(있는 경우),
    상승으로 분류할 키워드가 하나도 없으면 그 사실을 알리는 문장, 하락 폭이
    가장 큰 키워드(있는 경우에만 문장 추가, 없으면 문장 자체를 만들지 않음),
    보합(증감 폭이 정확히 0이거나 `searchTrend`가 "flat")인 키워드가 있으면
    그 개수, 계산 불가한 값이 있는 키워드가 있으면 그 사실, 그리고 이 지표가
    검색 관심도·추세일 뿐 실제 판매량·매출이 아니라는 마무리 문장입니다.
  - **동점 처리**: 값이 같으면 먼저 선택한 키워드가 우선합니다(선택 순서
    기준). 증감 폭이 정확히 0이면 보합으로 분류합니다.
  - **키워드를 1개만 선택한 경우**: "다른 키워드와 비교"하는 표현 대신, 그
    키워드 하나의 검색 관심도 지수와 증감 폭만 설명하는 문장으로 바뀝니다.
  - **선택한 키워드가 없는 경우**: 요약 문장 입력창이 비고, "핵심 키워드 또는
    상승세 키워드를 선택하면 비교 요약 문장이 생성됩니다."라는 안내 문구가
    자리 안내(placeholder)로 표시됩니다. [다시 생성]/[요약 문장 복사] 버튼도
    비활성화됩니다.
- **직접 수정 가능**: 요약 문장은 읽기 전용이 아니라 실제 `<textarea>`라서 직접
  고쳐 쓸 수 있습니다. [다시 생성]을 누르면 직접 고친 내용을 버리고 지금 데이터로
  새로 만듭니다.
- **클립보드 복사**: [요약 문장 복사]를 누르면 입력창에 지금 있는 내용(사용자가
  고친 내용이 있으면 그 내용 그대로) 그대로를 클립보드에 복사합니다.
  `navigator.clipboard.writeText`를 먼저 시도하고, 이 API를 쓸 수 없거나
  실패하면 숨김 `<textarea>` + `document.execCommand("copy")` 방식으로 자동
  전환합니다(기존 "결과 요약 복사" 기능과 같은 방식 재사용,
  `copyTextToClipboard`). 복사할 내용이 없으면 "복사할 요약 문장이 없습니다."를
  보여주고 복사를 시도하지 않으며, 버튼을 빠르게 여러 번 눌러도 복사 요청이
  겹치지 않도록 막았습니다.
- **안내 문구 갱신은 한 곳(`aria-live="polite"`)**: CSV 다운로드/복사 성공·실패
  안내는 같은 상태 영역에 표시되며, 이전 메시지를 쌓지 않고 새 메시지로
  교체합니다. 색만으로 성공/실패를 구분하지 않고 문구로도 알려줍니다.
- **`server.js`는 전혀 관여하지 않습니다**: CSV 생성, 요약 문장 생성, 클립보드
  복사 모두 브라우저(`index.html`)에서만 일어나는 동작이며, 새 API 라우트나
  서버 쪽 저장은 추가하지 않았습니다.

### 애니메이션 주간 관심도 랭킹 (v3.9)

"2. 애니메이션 트렌드" 화면에서, 최대 5개까지 등록한 애니메이션의 네이버 검색
관심도를 주간 단위로 비교합니다. 서버 쪽 API(`POST /api/anime-weekly-trends`)는
v3.9 1단계에서 먼저 만들고 실제 네이버 호출까지 검증했으며, 이 화면(2단계)은 그
API 응답만 그대로 그립니다 — 화면에서 순위나 지수를 다시 계산하거나 새로 만들어
내지 않습니다.

- **"순위"의 의미**: 이 순위는 **네이버 공식 인기 애니메이션 순위가 아니라, 지금
  등록한 애니메이션끼리만 비교한 상대 순위**입니다. 검색 관심도 지수도 실제 검색
  횟수·판매량·매출을 뜻하지 않습니다. 비교 대상 구성이 바뀌면(작품을 추가/삭제하면)
  지수의 정규화 기준 자체가 달라질 수 있습니다.
- **애니메이션 설정**: 대표 작품명 1개 + 관련 검색어(한글명·영문명·줄임말 등,
  쉼표 또는 줄바꿈으로 구분) 조합을 그룹당 하나씩, 최소 1개~최대 5개까지 등록할 수
  있습니다. [작품 추가]/각 그룹의 [삭제] 버튼으로 그룹 수를 조절하고, [주간 랭킹
  조회]를 눌러야만 서버에 요청이 나갑니다(입력 중 Enter나 타이핑만으로는 요청이
  발생하지 않습니다).
  - **시작 목록**: 저장된 목록이 없는 최초 접속에서는 원피스·귀멸의 칼날·주술회전·
    진격의 거인·나루토 5개 작품이 입력 폼에 미리 채워집니다. **이 시작 목록은 순위
    결과가 아니라, 기능을 바로 시험해 볼 수 있도록 미리 채워둔 예시 비교 목록일
    뿐입니다.** 실제 순위·지수는 이 목록을 그대로 조회했을 때 서버가 돌려주는
    값으로만 정해집니다.
  - **[시작 목록 복원]**은 **입력 폼만** 시작 목록으로 되돌립니다. 이미 저장된
    목록(`trendTop10.animeWatchlist`)이나 화면에 표시된 순위 결과는 지우지 않으며,
    바뀐 내용을 실제로 반영하려면 [주간 랭킹 조회]를 다시 눌러야 합니다.
- **조회 기간(weeks)**: 최근 4·8·12·16·26주 중 선택할 수 있고 기본값은
  **12주**입니다. 서버에는 항상 숫자로 전달되며, 화면 순위는 서버가 계산한 "가장
  최근에 끝난 완료 주" 기준입니다(진행 중인 이번 주는 집계에서 제외됩니다).
- **브라우저 저장(`trendTop10.animeWatchlist`)**: 조회에 **성공**했을 때만, 서버
  응답 `items`의 `title`/`keywords`만 정규화해서 이 브라우저에 저장합니다. 예:
  ```json
  [{ "title": "귀멸의 칼날", "keywords": ["귀멸의 칼날", "귀멸", "Demon Slayer"] }]
  ```
  **순위·지수·ratio·캐시 여부 같은 조회 결과 자체는 저장하지 않습니다.** 실패한
  요청은 기존 저장값을 덮어쓰지 않고, 저장이 실패해도(브라우저 저장 차단 등) 조회
  결과 자체는 정상적으로 표시됩니다(이번 세션 동안만 정상 사용). 저장된 값이
  손상되어 있으면(JSON 깨짐, 구조 오류 등) 자동으로 지우고 시작 목록으로
  복구합니다. 페이지를 열면 저장된 목록(없으면 시작 목록)으로 **최초 1회만**
  자동 조회합니다.
- **주간 핵심 요약**: 이번 주 관심도 1위, 전주 대비 최고 상승(지수 포인트 기준,
  "%"가 아니라 항상 **"p"** 단위), 최근 4주 평균 1위, 집계 기준(집계 주·조회
  기간·비교 작품 수·캐시 여부)을 카드 4장으로 보여줍니다. 값을 계산할 수 없으면
  "집계 불가" 또는 "상승으로 분류할 작품이 없습니다."라고만 표시합니다.
- **주간 관심도 순위표**: 서버가 이미 계산해 응답한 `currentRank` 순서 그대로
  표시할 뿐, 화면에서 순위를 다시 계산하지 않습니다. `currentRank`가 없는 항목은
  뒤로 보내고, 1~3위는 기존 가격순위 배지(`.price-rank-badge`)와 같은 모양의
  작은 배지로 강조합니다. 지수 변화는 ▲/▼ 기호와 함께 항상 "p" 단위로, 순위
  변화는 ▲/▼/"— 유지"/"신규"/"계산 불가"로 표시합니다.
- **피규어 상품 검색 연결**: 순위표의 각 행에 "{작품명} 피규어 상품 보기" 버튼이
  있습니다. 누르면 새 쇼핑 검색 API를 만들지 않고 **기존 "3. 상품 검색" 기능을
  그대로 재사용**해서 "{작품명} 피규어"로 검색하고, 상품 검색 섹션으로 화면을
  이동시킵니다. 이 검색이 실패해도 애니메이션 순위 결과는 그대로 유지됩니다(서로
  다른 상태를 쓰기 때문입니다). 검색 결과 개수는 판매량이나 수요가 아닙니다.
- **최근 8주 검색 관심도 추이**: 외부 차트 라이브러리 없이 인라인 SVG로 그립니다.
  세로축은 0~100 검색 관심도 지수, 가로축은 주간 집계 기준일이며, 결측(데이터
  없음) 구간은 0으로 바꾸지 않고 선을 끊어서 표시합니다. 조회 기간이 8주보다
  길어도 그래프는 항상 최근 완료 8주만 보여주고(4주만 조회했으면 4주만), 범례
  버튼(`aria-pressed`)으로 작품별 표시/숨김을 전환할 수 있습니다. 같은 데이터를
  그래프 아래 접근 가능한 표로도 제공해서, 포인트 하나하나에 마우스/키보드로
  포커스하지 않아도 스크린 리더로 같은 정보를 확인할 수 있게 했습니다.
- **주간 랭킹 CSV 다운로드**: 기존 "비교 결과 CSV" 헬퍼(`escapeCsvCell`,
  `protectCsvFormula`, UTF-8 BOM, RFC4180 이스케이프)를 그대로 재사용합니다. 열
  구성은 `순위,작품,이번주_검색관심도지수,전주_검색관심도지수,지수변화_포인트,
  순위변화,최근4주평균지수,평균계산주수,상태,집계주,조회시작일,조회종료일`이며,
  파일명은 `KOO_애니메이션주간랭킹_YYYYMMDD_HHmm.csv` 형식입니다. 조회 결과가
  없으면 다운로드를 막고 안내만 보여줍니다.
- **오류 처리**: 조회 중에는 버튼이 비활성화되고 "애니메이션 주간 관심도를
  조회하는 중입니다."라고 안내합니다. 서버 오류 메시지(권한 문제, 호출 한도
  초과, 일시적 오류 등)는 `aria-live="polite"` 영역에 사용자용 문구로만
  표시하며, 네이버 원본 오류나 내부 오류 스택은 노출하지 않습니다. 실패해도
  화면에 남아 있던 직전 순위·그래프는 지우지 않습니다.
- **서버 쪽 캐시**: `/api/anime-weekly-trends`는 같은 조건(작품 구성·검색어·조회
  기간)이면 30분 동안 서버 메모리 캐시를 사용합니다. 이 캐시는 **서버 메모리에만
  저장되며, 서버를 재시작하면 그대로 사라집니다**(파일이나 DB에 남지 않습니다).

### 시장현황 키워드 직접 입력하기 (v3.4)와 자동 저장(v3.5)

"1. 시장현황" 영역 안의 "시장현황 키워드" 카드에서 조회 키워드를 직접 바꿀 수
있습니다.

- 여러 줄 입력창에 키워드를 **줄바꿈 또는 쉼표(,)로 구분**해서 입력한 뒤
  **[키워드 적용]**을 누르면, 그 키워드로 시장현황을 다시 조회합니다.
  ```
  선크림
  쿠션
  립틴트
  ```
  또는
  ```
  선크림, 쿠션, 립틴트
  ```
- 키워드 앞뒤 공백은 자동으로 지워지고, 빈 값과 중복 키워드는 정리됩니다(입력
  순서는 유지).
- **최대 10개, 키워드 1개당 최대 30자**까지 입력할 수 있습니다. 이 범위를
  벗어나면 서버가 요청을 거부하고 이해할 수 있는 오류 메시지를 보여줍니다.
- **[기본값 복원]**을 누르면 저장된 키워드가 지워지고, 입력창과 시장현황이
  서버에 설정된 기본 키워드 10개로 돌아갑니다.
- 키워드를 입력하지 않고 그대로 두면 기존과 동일하게 기본 키워드로 동작합니다.

**v3.5부터는 [키워드 적용]이 성공한 키워드가 이 브라우저에 자동으로
저장됩니다.**

- 저장 키 이름: **`trendTop10.marketStatusKeywords`** (브라우저
  `localStorage`, 값은 키워드 문자열 배열을 JSON으로 바꾼 것뿐입니다. 예:
  `["선크림","쿠션","립틴트"]`)
- **저장되는 건 키워드 배열뿐**입니다. 검색 결과, 시장현황 점수, API
  키/Secret, 오류 내용 등 다른 데이터는 저장하지 않습니다.
- 같은 브라우저로 **다음에 다시 접속하거나 새로고침**하면, 저장된 키워드를
  자동으로 불러와 시장현황을 다시 조회하고 입력창/"현재 적용 키워드" 문구를
  그 값으로 채웁니다.
- 저장은 이 **브라우저·기기 안에서만** 유지됩니다(`localStorage`는 브라우저별·
  기기별로 독립적이라 다른 브라우저나 다른 기기에는 남지 않습니다).
- 시크릿 모드 등으로 브라우저 저장이 막혀 있으면 저장 자체가 조용히
  건너뛰어지고, **이번 접속(탭을 닫기 전까지) 동안만** 적용된 키워드로
  동작합니다 — 페이지 기능은 이 경우에도 정상적으로 계속 작동합니다.
- 키워드는 시장현황을 조회하기 위해 요청마다 서버(`server.js`)에 **전달되지만,
  서버 파일이나 `.env`에 영구 저장되지는 않습니다.** 서버는 매 요청마다 새로
  검증하고 네이버 API를 조회할 뿐, 키워드 자체를 어디에도 남기지 않습니다.
- 저장된 키워드로 자동 재조회했는데 서버가 400(더 이상 유효하지 않음)으로
  거부하면, 저장값을 지우고 기본 키워드로 한 번만 다시 조회한 뒤 짧은 안내를
  보여줍니다. 네트워크 오류나 일시적인 서버 오류(500/502/503 등)에서는 저장값을
  그대로 두고, 다음에 다시 시도해볼 수 있게 합니다.
- 키워드 카드 아래 "저장 상태" 문구(이 브라우저에 저장됨 / 서버 기본값 사용 중 /
  저장되지 않음)로 지금 상태를 바로 확인할 수 있습니다.

## 주의사항

- **구매순위·판매순위·네이버 공식 인기순위가 아닙니다.** 시장현황과 상품 검색 결과
  모두 참고용입니다.
- API 호출이 실패하면 예시(샘플) 데이터 또는 직전 조회 결과가 화면에 표시될 수
  있습니다.
- **네이버 API 응답 원본은 파일/DB에 저장하지 않습니다.** 서버는 요청이 들어올
  때마다 네이버 API를 새로 호출해서 그때그때 응답을 만들 뿐, 결과를 어디에도
  남기지 않습니다.
- 브라우저 `localStorage`에는 **최근 검색어 텍스트만** 저장됩니다. 상품명·가격·시장
  현황 등 API 응답 데이터는 저장하지 않습니다. [기본 검색어로 되돌리기] 버튼을
  누르면 저장된 값도 함께 지워집니다.
- API 키(Client ID)/Secret은 `index.html`(브라우저/클라이언트 코드)에 절대 넣지
  않고, 반드시 서버(`server.js`) 쪽 환경변수(`.env` 파일 또는 Render 환경변수)로만
  관리합니다.
- Render 무료 플랜은 오랫동안 방문이 없으면 서버가 잠들었다가, 다시 접속할 때 첫
  로딩이 몇십 초 정도 느릴 수 있습니다. 정상 동작이니 잠시 기다렸다가 새로고침해
  주세요.

## 파일 구조

- `index.html`: 실제 서비스 화면. HTML/CSS/JavaScript가 모두 이 파일 하나에 들어
  있으며, API 키/Secret은 담고 있지 않습니다. 헤더 장식용 이모티콘은
  `url("/assets/emoticons.png")`로 서버에서 불러옵니다(v3.3부터 base64 인라인

  시점 지시를 지키기 위해 이미지를 base64로 인코딩해서 `index.html`의 CSS 안에
  통째로 넣어뒀었고, 그 결과 `index.html` 파일 크기가 약 65KB에서 1.3MB
  가까이 늘어났습니다.
- **`server.js`에 `/assets/emoticons.png` 라우트 추가**: `GET /` 라우트 바로
  다음에 `pathname === "/assets/emoticons.png"`인 요청만 정확히 문자열로
  비교해서 처리합니다. `assets` 폴더 전체를 여는 범용 정적 파일 서버가 아니라
  이 파일 하나만 고정 경로(`path.join(__dirname, "assets", "emoticons.png")`)로
  읽어서 내려주므로, URL에 `../`를 아무리 넣어도 다른 경로와 문자열이 달라져
  이 라우트에 걸리지 않고 그냥 404로 떨어집니다(경로 조작으로 다른 파일을 읽을
  수 없음). 성공하면 `Content-Type: image/png`와 캐시 헤더를 붙여 이미지
  바이트를 그대로 응답하고, 파일을 찾지 못하면 404를 반환합니다. 기존 라우트
  (`/`, `/api/health`, `/api/naver-ready`, `/api/market-status`,
  `/api/naver-shopping-search`, 테스트용 API들)는 전혀 손대지 않았습니다.
- **`index.html`에서 base64 제거**: CSS의 `background-image`를
  `url("/assets/emoticons.png")`로 바꾸고, 1.2MB가 넘던 base64 문자열은
  완전히 삭제했습니다. 그 결과 `index.html` 파일 크기가 다시 약 79KB 수준으로
  줄었습니다. 이모티콘 클래스(`emoji-happy` 등 10종)와 PC 10개/태블릿 6개/
  모바일 3개 반응형 배치, 제목이 가려지지 않는 z-index 구조는 그대로
  유지했습니다.
- **주의**: 이 방식은 `index.html`을 `node server.js`로 띄운 서버(또는 Render
  배포 주소)를 통해 열 때만 이모티콘 이미지가 보입니다. `index.html`을
  더블클릭해서 `file://`로 직접 열면(서버 없이) 이모티콘 이미지 요청이 실패
  하지만, 흰 원 배지만 남고 레이아웃이 깨지지는 않습니다. 다른 기능(시장현황,
  상품 검색 등)도 원래부터 서버가 있어야 동작하므로 실사용에는 영향이
  없습니다.
- **백업**: `backups/index_v3.3.html`, `backups/server_v3.3.js` 생성.
- `package.json` 버전 표기를 `3.3.0`으로 갱신.

API 키/Secret은 이번에도 어디에도 노출되지 않았고(`/assets/emoticons.png`
라우트는 이미지 파일만 다룹니다), `.env` 파일과 기존 네이버 API 라우트들은
그대로 유지됩니다.

### v3.4 — 시장현황 키워드 직접 입력 기능 추가

**이번 버전은 v3.0~v3.3과 달리 실제 기능이 추가된 버전입니다.** "1. 시장현황"이
쓰는 관심 키워드를 화면에서 직접 입력·적용할 수 있게 했습니다. 기존 시장현황·
상품 검색·상세 결과·사용 안내 기능, 헤더 이모티콘과 반응형 배치, 기존 네이버 API
라우트와 테스트 API는 모두 그대로 유지됩니다.

- **`index.html` — "시장현황 키워드" 카드 추가**: "1. 시장현황" 영역 안에 여러
  줄 입력창(`textarea`)과 **[키워드 적용]**/**[기본값 복원]** 버튼, "현재 적용
  키워드" 문구를 담은 카드를 새로 추가했습니다. 입력창에는 줄바꿈 또는 쉼표(,)로
  구분해서 키워드를 입력할 수 있고, 앞뒤 공백 제거·빈 값 제외·중복 제거(입력
  순서 유지)는 화면에서 먼저 한 번 정리한 뒤 서버로 보냅니다.
- **요청 방식**: [키워드 적용]을 누르면 `URLSearchParams`로 안전하게 인코딩한
  같은 이름의 `keyword` 파라미터를 반복해서 요청합니다(문자열을 직접 이어붙이지
  않음). 예: `GET /api/market-status?keyword=선크림&keyword=쿠션&keyword=립틴트`.
  요청 중에는 두 버튼을 모두 비활성화하고 버튼 문구를 "적용 중..."으로 바꿔
  중복 요청을 막습니다.
- **`server.js` — `/api/market-status` 확장**: 기존 라우트를 그대로 두고,
  `requestUrl.searchParams.getAll("keyword")`로 반복된 `keyword` 파라미터를
  읽도록 확장했습니다. `keyword` 파라미터가 전혀 없으면(기존 방식) 서버에 설정된
  기본 키워드(`MARKET_STATUS_KEYWORDS`, 10개)를 그대로 사용합니다. 있으면
  검증(`validateMarketStatusKeywords`) 후 사용합니다: 앞뒤 공백·제어문자 제거,
  빈 값·중복 제거(입력 순서 유지), 최대 10개, 키워드당 최대 30자. 검증을 통과하지
  못하면 HTTP 400과 함께 `{status, message, detail}` 형태의 JSON 오류를
  반환합니다(API 키나 내부 오류 정보는 포함하지 않음, 기존 라우트들과 동일한
  오류 응답 형식).
- **응답 필드 추가**: 기존 `keywords` 필드(서버에 설정된 기본 키워드 10개, 값이
  바뀌지 않음)는 그대로 두고, **`effectiveKeywords`**(이번 요청에 실제로 사용된
  키워드 배열)와 **`usesDefaultKeywords`**(기본 키워드 사용 여부) 두 필드를 새로
  추가했습니다. `index.html`은 "현재 적용 키워드" 문구를 표시할 때 이
  `effectiveKeywords`를 기준으로 삼습니다.
- **저장하지 않음**: 사용자가 입력한 키워드는 서버 파일이나 `.env`에 저장하지
  않습니다. 요청이 들어올 때마다 새로 검증하고 조회할 뿐이며, 페이지를
  새로고침하면 다시 서버 기본 키워드로 시작합니다.
- **보안**: 키워드를 화면에 표시할 때 `innerHTML`이 아니라 `textContent`만
  사용해서, 입력값이 HTML로 해석되지 않습니다. 사용자 입력을 파일 경로나
  명령어로 사용하는 곳은 없습니다. API 키/Client ID/Secret은 이번에도 어떤
  응답에도 포함되지 않습니다.
- **백업**: `backups/index_v3.4.html`, `backups/server_v3.4.js` 생성.
- `package.json` 버전 표기를 `3.4.0`으로 갱신. (`package-lock.json`은 이
  프로젝트에 존재하지 않아 별도로 맞출 파일이 없습니다.)

외부 라이브러리/CDN은 추가하지 않았고, API 키/Secret은 이번에도 `index.html`이나
응답 어디에도 들어가지 않았으며, `.env` 파일은 건드리지 않았습니다.

### v3.5 — 시장현황 키워드 브라우저 저장(localStorage) 기능 추가

**이번 버전도 실제 기능이 추가된 버전입니다.** v3.4에서 만든 "시장현황 키워드"
입력 기능 위에, 성공적으로 적용한 키워드를 이 브라우저에 저장해서 다음
접속/새로고침 때 자동으로 복원하는 기능을 더했습니다. `server.js`는 이번에는
수정할 필요가 없어서 **건드리지 않았습니다**(아래 백업은 현재 코드 기준으로
새로 생성).

- **저장 시점과 대상**: [키워드 적용]으로 요청한 **`/api/market-status`가
  성공하고, 그 응답의 `usesDefaultKeywords`가 `false`일 때만**(=서버가 실제로
  사용자 키워드를 적용했을 때만) 서버가 돌려준 `effectiveKeywords`를
  `localStorage`에 저장합니다. 사용자가 입력한 원본 텍스트가 아니라 서버가
  검증·정리해서 실제로 사용한 값을 저장합니다. 저장 키는
  `trendTop10.marketStatusKeywords`, 값은 키워드 배열을 JSON 문자열로 바꾼
  것뿐입니다. 요청이 실패하면 기존 저장값을 건드리지 않습니다.
- **자동 복원**: 페이지를 열 때 저장값을 읽어 JSON 파싱 → 클라이언트 검증까지
  마친 뒤(`readSavedMarketStatusKeywords`), 유효하면 그 키워드로
  `/api/market-status`를 자동 요청하고 입력창·"현재 적용 키워드" 문구를 서버
  응답의 `effectiveKeywords`로 채웁니다. 저장값이 없거나 애초에 무효했으면
  기존처럼 파라미터 없이 기본 키워드로 요청합니다.
- **400 거부 시 1회 재시도**: 저장된 키워드로 자동 재조회했는데 서버가
  400(키워드 검증 실패)으로 거부하면, 저장값을 지우고 기본 키워드로 **한 번만**
  다시 조회합니다(무한 재시도 없음). 짧은 안내("저장된 시장현황 키워드가 더 이상
  유효하지 않아 기본 키워드로 되돌렸습니다.")를 보여주되 API 키나 내부 오류
  정보는 표시하지 않습니다. 반대로 네트워크 오류나 500/502/503처럼 일시적일 수
  있는 오류에서는 저장값을 그대로 둡니다.
- **저장값 검증 (`validateStoredMarketStatusKeywords`)**: `localStorage`에서
  읽은 값은 신뢰하지 않고 항상 검증합니다 — 배열이어야 함, 모든 값이 문자열이어야
  함, 제어문자 제거, 앞뒤 공백 제거, 빈 값 제외, 중복 제거(순서 유지), 최대
  10개, 키워드당 최대 30자. 검증에 실패하면 저장값을 지우고 기본 키워드로
  복구합니다. textarea 원문을 다루는 기존 `parseMarketStatusKeywordInput`과는
  역할을 분리했습니다(하나는 "지금 입력한 문자열"을, 다른 하나는 "이미 저장돼
  있던 배열"을 다룸).
- **버튼 동작 확장**: [키워드 적용] 성공 후 저장에 성공하면 "키워드를 이
  브라우저에 저장했습니다.", 저장에 실패하면(브라우저 정책·저장 공간 문제 등)
  "시장현황은 적용했지만 브라우저에 저장하지 못했습니다."를 보여주되 시장현황
  조회 결과 자체는 그대로 유지합니다. [기본값 복원]은 저장값을 지우고 기본
  키워드를 다시 조회한 뒤 "서버 기본 키워드로 복원했습니다."를 보여줍니다.
- **저장 상태 문구**: 키워드 카드에 "저장 상태: 이 브라우저에 저장됨 / 서버

- **`localStorage` 안전성**: `readSavedMarketStatusKeywords` /

  영구 저장되지는 않는다"는 정확한 문구를 추가했습니다("서버에 전혀 전송되지

### v3.7 — 선택 키워드 비교 기능 추가

**이번 버전은 실제 기능이 추가된 버전입니다.** v3.6에서 만든 "핵심 키워드"·
"상승세 키워드" 카드의 키워드 칩을 선택 가능한 버튼으로 바꾸고, 선택한 키워드들의
실제 지표(검색 관심도 지수·증감 폭)를 비교하는 패널을 추가했습니다. 자세한 사용법은
위 "선택 키워드 비교 (v3.7)" 절을 참고하세요. `server.js`는 **수정하지
않았습니다**(기존 `/api/market-status` 응답만으로 구현했고, `diff`로 v3.6 백업과
완전히 동일함을 확인).

- **키워드 칩 → 선택 버튼**: `renderKeywordChips`가 만들던 표시 전용 `<span>`을
  `<button type="button">`으로 바꿨습니다. `data-keyword` 속성과 `aria-pressed`로
  선택 상태를 나타내고, 클릭(또는 키보드 Enter/Space, 버튼 기본 동작)으로
  토글합니다.
- **최대 5개 제한**: `selectedComparisonKeywords` 배열(선택 순서 유지)의 길이가
  5 이상이면 새 선택을 막고 안내만 보여줍니다. 이미 선택된 키워드를 해제하면
  다시 새로 선택할 수 있습니다.
- **핵심/상승세 카드에 동시에 있는 키워드**: 칩의 `data-keyword`(키워드 이름)를
  기준으로 선택 상태를 관리하므로, 같은 키워드가 양쪽에 있어도 하나의 선택으로
  취급되고 두 칩 모두 항상 같은 선택 상태로 보입니다
  (`syncComparisonChipStates`).
- **기본 선택/자동 정리 규칙**: 시장현황 데이터가 갱신될 때마다
  (`applyComparisonSelectionDefaults`) 새 `/api/market-status` 응답의
  `keywordData`에 더 이상 없는 선택 키워드를 제거합니다. 그 결과 선택이 0개가
  되면(처음 열었을 때 포함) 새 핵심 키워드 1번째 항목을 기본 선택합니다.
- **정렬**: 선택 순서 / 관심도 높은 순(`searchInterestScore` 내림차순) / 상승
  폭 높은 순(`searchTrendDiff` 내림차순) / 변동 폭 큰 순(`|searchTrendDiff|`
  내림차순) 4종. 동점이면 선택 순서 → 키워드명 순으로 정렬해서 매번 같은
  결과가 나오게 했습니다. 정렬은 표시 순서만 바꾸고, 실제 선택 배열의 순서는
  그대로 둡니다.
- **그래프**: 외부 라이브러리 없이 순수 HTML/CSS 막대로 구현했습니다(막대 길이는
  `style="width:...%"`로 직접 계산). "검색 관심도 지수"는 0~100 기준 가로
  막대이고, 값이 100을 넘거나 음수여도(원칙적으로는 없어야 하지만) 텍스트는
  원본을 보여주되 막대 길이만 0~100으로 clamp합니다. "검색 추세 증감 폭"은
  0을 가운데로 양수는 오른쪽, 음수는 왼쪽으로 뻗는 막대이며, 선택 항목 중
  절댓값이 가장 큰 값을 기준으로 스케일을 잡고, 유효한 값이 하나도 없으면
  0으로 나누지 않도록 스케일을 1로 고정합니다. 두 그래프 모두
  `aria-hidden="true"`로 장식 처리하고, 같은 데이터를 캡션 문장과 아래 표로
  다시 제공합니다.
- **버그 수정 하나**: 개발 중 "변동 폭 큰 순" 정렬에서, 값이 없는(`null`/
  `Infinity`) 키워드가 절댓값 계산 때문에 오히려 맨 위로 오는 문제를 실제 단위
  테스트로 발견해서 `safeAbsForSort` 함수를 추가해 바로잡았습니다(무효한 값은
  절댓값 기준으로도 항상 맨 뒤로 감).
- **키워드 상세 표**: API에 있는 필드만 사용합니다 — 키워드, 검색 관심도
  (`searchInterestScore`), 추세(`searchTrend`: up→상승/down→하락/flat→보합/
  그 외→확인 불가), 증감 폭(`searchTrendDiff`, %p). 판매량·매출·평점처럼
  API에 없는 값은 추가하지 않았습니다.
- **저장 범위**: 비교 선택 상태는 `localStorage`에 저장하지 않습니다. 새로고침하면
  기본 선택으로 돌아갑니다. 기존 `trendTop10.marketStatusKeywords`(적용 키워드
  저장/자동 복원)는 이번 기능과 완전히 분리되어 있고 그대로 동작합니다.
- **디자인**: v3.6의 민트·청록 카드 언어를 그대로 재사용했습니다. 선택된 칩은
  진한 청록 채움 + 흰 글자, 기본 칩은 연한 민트 배경, hover는 조금 더 진한
  민트입니다. 그래프 막대는 관심도=청록(`--color-primary`), 증감=상승/하락
  전용 색(`--color-positive`/`--color-negative`)을 그대로 재사용했고, 새로
  만든 변수는 없습니다.
- **백업**: `backups/index_v3.7.html`, `backups/server_v3.7.js`(내용은 v3.6과
  동일) 생성.
- `package.json` 버전 표기를 `3.7.0`으로 갱신. (`package-lock.json`은 이번에도
  존재하지 않아 별도로 맞출 파일이 없습니다.)

외부 라이브러리/CDN은 추가하지 않았고, 가짜 숫자나 `Math.random()`은 쓰지

### v3.8 — 비교 결과 CSV 다운로드·요약 문장 생성 기능 추가

**이번 버전도 실제 기능이 추가된 버전입니다.** v3.7에서 만든 "선택 키워드 비교"
패널 아래에, 선택한 키워드의 비교 결과를 CSV로 내려받거나 자동 생성된 요약
문장을 복사할 수 있는 "비교 결과 활용" 영역을 추가했습니다. 자세한 사용법은 위
"비교 결과 CSV 다운로드·요약 문장 (v3.8)" 절을 참고하세요. `server.js`는
**수정하지 않았습니다**(`diff`로 v3.7 백업과 완전히 동일함을 확인 — 새 API
라우트, 응답 형식 변경, 서버 쪽 CSV/요약 저장 모두 없음).

- **CSV 다운로드**: `getSelectedComparisonDataInSelectionOrder()`(정렬
  드롭다운과 무관하게 항상 선택 순서를 돌려주는 새 공용 헬퍼)로 데이터를 모은
  뒤 `buildComparisonCsv()`로 CSV 문자열을 만들고, `downloadComparisonCsv()`가
  UTF-8 BOM을 붙여 `Blob`으로 감싸 임시 `<a download>` 클릭으로 즉시
  다운로드시킵니다. 사용한 objectURL과 `<a>` 요소는 클릭 직후
  `URL.revokeObjectURL`/`removeChild`로 바로 정리해서 DOM에 잔여물을 남기지
  않습니다.
- **CSV 안전 처리**: `escapeCsvCell()`이 쉼표·큰따옴표·줄바꿈이 있는 값을
  RFC4180 규칙대로 이스케이프하고, `protectCsvFormula()`가 `=`/`+`/`-`/`@`/
  탭/캐리지리턴으로 시작하는 키워드 값 앞에 작은따옴표를 붙여 스프레드시트
  수식 주입(Formula Injection)을 막습니다. 이 방어는 사용자가 입력한 키워드
  문자열에만 적용하고, 음수를 나타낼 수 있는 실제 숫자 필드(검색 관심도 지수,
  증감 폭)에는 적용하지 않아 정상적인 음수 값이 텍스트로 오염되지 않습니다.
- **선택 순서와 정렬 순서 분리**: 기존 `getSortedComparisonItems()`(화면
  그래프·표용, 정렬 드롭다운을 따름)를 리팩터링해서 새로 만든
  `getSelectedComparisonDataInSelectionOrder()`를 내부적으로 재사용하도록
  바꿨습니다. "키워드→실제 데이터 매핑, 존재하지 않는 키워드 제외" 로직이
  두 곳에 중복되지 않고 한 곳에만 있으며, CSV/요약 문장은 항상 이 공용
  헬퍼만 사용해서 정렬 드롭다운 상태와 완전히 독립적으로 동작합니다.
- **요약 문장 생성**: `generateComparisonSummaryText()`가 선택된 키워드
  개수에 따라 다른 문장을 조립하는 순수 함수입니다(외부 AI 호출 없음, DOM
  없이도 단위 테스트 가능). 가장 높은 검색 관심도(`findItemWithHighestValidValue`),
  상승 폭이 가장 큰 키워드(`findItemWithMaxPositiveDiff`, 0보다 큰 값만
  후보), 하락 폭이 가장 큰 키워드(`findItemWithMinNegativeDiff`, 0보다 작은
  값만 후보)를 각각 찾고, 동점이면 선택 순서상 먼저 나온 키워드가
