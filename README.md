# 트렌드 TOP 10 (v3.2)

관심 키워드의 시장현황(네이버 데이터랩)과, 검색어로 직접 조회하는 네이버 쇼핑 검색
결과 TOP10을 함께 보여주는 **개인 참고용** 대시보드입니다. **공식 순위 서비스가
아니라, 네이버 데이터를 화면용으로 정리해서 보여주는 개인 참고·연습 프로젝트**라는
점을 꼭 기억해 주세요.

v3.0은 새 기능을 추가하는 버전이 아니라, 지금까지 만든 기능을 그대로 유지하면서
화면 문구·화면 흐름·오류 안내·모바일 화면·README를 개인 실사용 기준으로 다시
정리한 **안정화 버전**이었습니다. v3.1은 그 위에서 **기능 추가 없이 디자인만
다시 다듬은 버전**이었고, v3.2는 v3.1 디자인 위에 **상단 헤더에 개인 캐릭터
이모티콘을 얹어 브랜딩 요소를 더한 버전**입니다. 이번에도 기능 변경은 없습니다.

## 이 프로젝트는 무엇인가요? (목적)

"트렌드 TOP 10"은 화면을 아래 순서로 살펴보는 개인 참고용 대시보드입니다.

1. **시장현황**: 미리 정해둔 관심 키워드 10개(선크림, 양산, 여름원피스, 쿨토시,
   제습기, 미니선풍기, 래쉬가드, 샌들, 냉감이불, 물놀이용품)의 최근 30일 검색·쇼핑
   클릭 추이를 네이버 데이터랩(검색어트렌드·쇼핑인사이트) API로 참고합니다.
2. **상품 검색**: 궁금한 검색어를 직접 입력하면 네이버 쇼핑 검색 API가 실제로
   돌려주는 검색 결과 TOP10(상품명·쇼핑몰·최저가·카테고리 등)을 조회합니다. 기본
   정렬은 정확도순(`sort=sim`)입니다.
3. **상세 결과**: 상품 검색 결과를 조회 기준·결과 요약 문장·가격 요약 카드·카테고리
   필터·순위 표로 자세히 살펴보고, 결과 요약을 텍스트로 복사할 수 있습니다.
4. **사용 안내**: 이 화면을 어떻게 이해하고 써야 하는지, 그리고 운영 점검 링크를
   마지막으로 정리해서 보여줍니다.

이 화면이 보여주는 모든 데이터는 **구매순위·판매순위·네이버 공식 인기순위가
아닙니다.** 빌드 도구나 외부 라이브러리 없이 `index.html` 파일 하나(+ 서버
`server.js`)로 동작합니다. API 연결에 실패하면 예시(샘플) 데이터가 화면에 표시될 수
있습니다.

## 로컬 실행 방법

`index.html`은 화면에 보여줄 데이터를 `server.js`(API 서버)에게 요청해서 받아옵니다.
그래서 아래 순서로 실행해야 정상적으로 데이터가 보입니다.

1. 이 폴더에서 터미널(PowerShell)을 열고 아래 명령어로 서버를 먼저 실행합니다.
   ```
   node server.js
   ```
   `네이버 쇼핑 검색 TOP10 서버 실행 중: http://localhost:3000/...` 메시지가 보이면
   성공입니다.
2. 서버를 켜둔 상태로 `index.html` 파일을 더블클릭해서 브라우저로 엽니다.

서버를 켜지 않고 `index.html`만 열면, "서버 연결 실패" 안내 배너가 화면에 표시되고
직전에 불러온 결과(또는 예시 데이터)가 그대로 남습니다.

로컬에서 실제 네이버 데이터를 보려면 아래 "환경변수 안내"에 따라 `.env` 파일에 API
키를 채워야 합니다. `.env` 파일이 없거나 값이 비어 있으면, 서버는 오류를 던지지 않고
예시 데이터로 자동 대체합니다.

## Render 배포 및 환경변수 안내

### 배포 설정값

[render.com](https://render.com)에서 "New +" → "Web Service"로 새 서비스를 만들 때
아래 값을 그대로 입력하면 됩니다.

| 설정 항목 | 입력할 값 |
| --- | --- |
| Service Type | **Web Service** |
| Language(Runtime) | **Node** |
| Branch | **main** |
| Build Command | `npm install` (또는 비워둬도 됩니다 — 이 프로젝트는 외부 패키지를 쓰지 않습니다) |
| Start Command | `node server.js` |

### 환경변수 채우기

네이버 API는 기능별로 따로 신청하기 때문에, 키가 **한 쌍**만 있을 수도 있고
**두 쌍**(용도별)일 수도 있습니다. 두 경우 모두 지원합니다.

- **키가 한 쌍인 경우** — 아래 두 값만 채우면 두 API(쇼핑인사이트·검색어트렌드) 모두에
  공용으로 사용됩니다.
  ```
  NAVER_CLIENT_ID
  NAVER_CLIENT_SECRET
  ```
- **키가 두 쌍인 경우** — 용도별 값이 있으면 공용 값보다 우선 사용됩니다.
  ```
  NAVER_SHOPPING_CLIENT_ID
  NAVER_SHOPPING_CLIENT_SECRET
  NAVER_SEARCH_TREND_CLIENT_ID
  NAVER_SEARCH_TREND_CLIENT_SECRET
  ```

로컬에서는 이 이름 그대로 `.env` 파일(`.env.example` 참고)에 채우고, Render에서는
"Environment Variables" 설정 화면에 같은 이름으로 등록합니다.

> ⚠️ **중요**: 실제 API 키/Secret 값은 GitHub 저장소(코드)나 `.env` 파일에 커밋해서
> 올리는 것이 아니라, 반드시 Render의 "Environment Variables" 설정 화면에 직접
> 입력해야 합니다. GitHub에는 `.env.example`처럼 값이 비어있는 안내 파일만 올라가야
> 하고, `.gitignore`가 `.env` 파일 자체를 Git에서 제외해줍니다. API 키/Secret은
> `index.html` 같은 브라우저(클라이언트) 코드에는 절대 넣지 않습니다 — 반드시
> `server.js` 쪽 환경변수로만 다룹니다.

### 설정이 잘 됐는지 확인하기

배포 후(또는 로컬에서) `/api/naver-ready` 주소에 접속하면 아래처럼 두 값이 각각
표시됩니다.

```json
{ "shoppingInsightReady": true, "searchTrendReady": true }
```

`false`가 보이면 해당 용도의 환경변수 값(또는 공용 `NAVER_CLIENT_ID`/`SECRET`)을 다시
확인해야 합니다. Render 로그(Logs) 탭에서 서버 실행 메시지가 뜨는지도 함께 확인하면
좋습니다.

## 사용 API 설명

이 대시보드는 네이버 오픈 API 중 아래 3가지를 사용합니다. 모두 `server.js`에서만
호출하며, `index.html`(브라우저)은 이 API들을 직접 호출하지 않습니다.

| API | 사용 위치 | 알려주는 것 |
| --- | --- | --- |
| 데이터랩 검색어트렌드 | `/api/market-status` (1. 시장현황) | 관심 키워드별 검색 관심도 추이 |
| 데이터랩 쇼핑인사이트 | `/api/market-status` (1. 시장현황) | 관심 키워드별 쇼핑 클릭 추이 |
| 쇼핑 검색 API | `/api/naver-shopping-search` (2/3. 상품 검색·상세 결과) | 검색어 기준 상품 검색 결과 TOP10 |

API 호출이 실패하면(키 미설정, 네이버 서버 오류, 응답에 쓸만한 데이터 없음 등) 각
영역은 독립적으로 예시(샘플) 데이터 또는 직전 결과로 대체되고, 화면에는 그 사실을
알리는 안내 배너가 표시됩니다.

## 검색순위와 가격순위 차이

"3. 상세 결과" 표에는 순위를 나타내는 열이 두 개 있는데, 서로 뜻이 다릅니다.

- **검색순위**: 네이버 쇼핑 검색 API가 **반환한 순서** 그대로입니다(기본 정렬:
  정확도순 `sort=sim`). 진한 파란 숫자로 표시됩니다.
- **가격순위**: **현재 화면에 표시된 결과 안에서** 최저가(`lprice`) 기준으로 낮은
  가격부터 매긴 참고 순위입니다. 파란 배지로 표시되며, 카테고리 필터를 바꾸면 그
  범위 안에서 다시 계산됩니다. 가격 정보가 없는 상품은 `-`로 표시됩니다.

**두 순위 모두 구매순위·판매순위·네이버 공식 인기순위가 아닙니다.** 표 아래 안내
문구와 "4. 사용 안내"에도 같은 설명이 유지됩니다.

## 시장현황의 의미

"1. 시장현황"은 서버(`server.js`)에 미리 정해둔 **관심 키워드 10개** 기준으로, 최근
30일 검색 관심도(검색어트렌드)와 쇼핑 클릭 추이(쇼핑인사이트)를 요약해서 보여주는
영역입니다.

- **네이버 공식 인기검색어 순위가 아니라, 설정한 관심 키워드 기준의 참고용
  정보**입니다.
- 관심도 점수는 네이버 데이터랩 API 호출(최대 5개 키워드씩 두 묶음)마다 각각
  0~100으로 새로 정규화되므로, 서로 다른 묶음에 속한 키워드끼리의 절대 점수 비교는
  근사치입니다.
- "변동" 값도 네이버가 제공하는 공식 값이 아니라, 최근 7일 평균과 이전 7일 평균의
  차이를 서버가 직접 계산한 근사치입니다.

## 주의사항

- **구매순위·판매순위·네이버 공식 인기순위가 아닙니다.** 시장현황과 상품 검색 결과
  모두 참고용입니다.
- API 호출이 실패하면 예시(샘플) 데이터 또는 직전 조회 결과가 화면에 표시될 수
  있습니다.
- **네이버 API 응답 원본은 파일/DB에 저장하지 않습니다.** 서버는 요청이 들어올
  때마다 네이버 API를 새로 호출해서 그때그때 응답을 만들 뿐, 결과를 어디에도
  남기지 않습니다.
- 브라우저 `localStorage`에는 **최근 검색어 텍스트만** 저장됩니다. 상품명·가격·시장
  현황 등 API 응답 데이터는 저장하지 않습니다. [기본 검색어로 되돌리기] 버튼을
  누르면 저장된 값도 함께 지워집니다.
- API 키(Client ID)/Secret은 `index.html`(브라우저/클라이언트 코드)에 절대 넣지
  않고, 반드시 서버(`server.js`) 쪽 환경변수(`.env` 파일 또는 Render 환경변수)로만
  관리합니다.
- Render 무료 플랜은 오랫동안 방문이 없으면 서버가 잠들었다가, 다시 접속할 때 첫
  로딩이 몇십 초 정도 느릴 수 있습니다. 정상 동작이니 잠시 기다렸다가 새로고침해
  주세요.

## 파일 구조

- `index.html`: 실제 서비스 화면. HTML/CSS/JavaScript가 모두 이 파일 하나에 들어
  있으며, API 키/Secret은 담고 있지 않습니다. 헤더 장식용 이모티콘은
  `assets/emoticons.png`를 base64로 인라인해서 CSS 안에 넣어뒀습니다(별도 이미지
  요청 없이 항상 표시됨).
- `assets/emoticons.png`: 헤더 장식용 캐릭터 이모티콘 10종이 담긴 2열 x 5행
  스프라이트 원본 이미지. `index.html`의 CSS가 이 파일 하나만을 원본으로 삼아
  캐릭터별로 잘라 씁니다.
- `server.js`: API 서버. 네이버 데이터랩·쇼핑 검색 API를 호출하고, 실패 시 예시
  데이터로 대체하는 로직을 담당합니다. `.env` 파일이 있으면 자동으로 읽어옵니다.
  정적 파일 서버 기능은 없으므로(`/`와 `/api/*`만 응답), 이미지 등 추가 자산은
  `index.html`에 인라인하는 방식을 씁니다.
- `.env.example`: 어떤 환경변수가 필요한지 보여주는 예시 파일 (실제 키 값은 없음)
- `.env`: 실제 API 키/Secret이 담긴 파일. **git에는 올라가지 않습니다**(`.gitignore`).
- `.gitignore`: `.env`, `node_modules/`, 로그·임시 파일이 git에 올라가지 않도록
  막는 설정 파일
- `package.json`: 프로젝트 메타 정보와 실행 스크립트(`npm start` → `node server.js`)
- `backups/`: 주요 버전 시점의 `index.html` / `server.js` 스냅샷을 남겨두는 폴더
  (최신 `index.html` 백업본: `index_v3.2.html`, 최신 `server.js` 백업본: `server_v3.0.js`)
- `CLAUDE.md`: Claude Code가 이 프로젝트에서 작업할 때 참고하는 안내 문서

### v3.0 — 개인 실사용 안정화 버전

새 기능 추가 없이, 아래 항목을 개인 참고용 기준으로 다시 점검·정리했습니다.

- **화면 문구**: "1. 시장현황 → 2. 상품 검색 → 3. 상세 결과 → 4. 사용 안내" 흐름이
  자연스럽게 읽히도록 안내 문구를 재점검. "3. 상세 결과" 영역 시작 부분에, 이 영역이
  "2. 상품 검색" 결과를 자세히 보여주는 곳이라는 안내 문장을 추가.
- **상세 결과 표**: 검색순위(네이버 쇼핑 검색 API 반환 순서)와 가격순위(현재 표시
  결과 안 최저가 기준)의 의미, 그리고 둘 다 구매순위·판매순위·공식 인기순위가
  아니라는 안내가 유지되고 있는지 확인.
- **오류/로딩 안내**: 시장현황·상품 검색 각각의 로딩·실패·fallback 안내, API 실패
  시 예시 데이터 표시 안내, Render 무료 플랜 첫 로딩 지연 안내가 모두 유지되고
  있는지 확인.
- **모바일 CSS**: 1~4번 섹션이 세로로 자연스럽게 쌓이는지, 순위 표의 검색순위·
  최저가·가격순위·[상품 보기] 버튼이 겹치지 않는지, 이미지 없음/가격 정보 없음
  상태도 깨지지 않는지 확인.
- **README**: 버전별 변경 이력을 나열하던 문서를 프로젝트 목적·실행 방법·API
  설명·주의사항 중심으로 재구성 (지금 보고 있는 이 문서).
- **백업**: `backups/index_v3.0.html`, `backups/server_v3.0.js` 생성.
- `package.json` 버전 표기를 `3.0.0`으로 갱신.

`index.html`과 `server.js`의 기존 기능(시장현황, 상품 검색, 상세 결과, 가격 요약
카드, 검색순위/가격순위, 카테고리 필터, 결과 요약 문장, 결과 요약 복사, fallback
동작, 조회 중/오류 UX, 사용 안내, 운영 점검 링크)은 모두 그대로 유지됩니다.

### v3.1 — 디자인 리뉴얼 (기능 변경 없음)

**이번 버전은 새 기능을 추가한 버전이 아니라, v3.0의 기능을 그대로 둔 채
`index.html`의 디자인/사용성만 다시 다듬은 버전입니다.** 화면 흐름("1. 시장현황 →
2. 상품 검색 → 3. 상세 결과 → 4. 사용 안내")과 위에 정리된 모든 기능은 동일하게
동작합니다. `server.js`는 수정하지 않았습니다.

- **색상 톤 정리**: 포인트 컬러를 파란색 하나(`--color-primary`)로 통일하고,
  배경·테두리·글자색을 중성 회색 톤 디자인 토큰(CSS 변수)으로 다시 정리했습니다.
  성공(초록)/주의(호박)/오류(빨강) 상태 색은 브랜드 컬러와 겹치지 않도록 고정
  토큰으로 분리했습니다.
- **헤더**: 단색 배경 대신 은은한 그라디언트를 적용하고, 기준 날짜(`#baseDate`)를
  알약 모양 배지로 다듬어 제목/부제/기준일의 위계를 더 명확히 했습니다.
- **화면 흐름 강조**: "1. 시장현황" · "2. 상품 검색" · "3. 상세 결과" · "4. 사용
  안내" 제목 앞에 원형 번호 배지(●1 ●2 ●3 ●4)를 붙여서 4단계 흐름이 한눈에
  보이도록 했습니다.
- **카드 스타일 통일**: 모든 섹션 카드, 시장현황/가격 요약 카드, 조회 기준 카드가
  같은 디자인 언어(테두리·모서리 반경·그림자·라벨/값 위계)를 쓰도록 정리했습니다.
  요약 카드는 라벨(작은 대문자)·값(굵은 큰 글씨) 위계를 명확히 하고 왼쪽에 포인트
  컬러 강조선을 추가했습니다.
- **버튼**: 주요 버튼(채움)과 보조 버튼([기본 검색어로 되돌리기], 테두리형)의
  구분을 명확히 하고, hover(색 변화 + 그림자)·disabled(회색조 처리) 상태를
  다듬었습니다. 카테고리 필터 버튼도 같은 언어(선택 시 채움, 비선택 시 테두리)로
  통일했습니다.
- **상세 결과 표**: 헤더를 옅은 배경 + 대문자 라벨로 정리하고, 행에 hover 강조를
  추가했습니다. **검색순위는 중성 잉크색 숫자, 가격순위는 파란 배지**로 색 계열을
  분리해서 두 순위가 서로 다른 개념이라는 점이 더 잘 드러나도록 했습니다. [상품
  보기] 버튼과 긴 상품명(줄바꿈·굵기) 가독성도 함께 다듬었습니다.
- **상태 배너/배지**: "네이버 데이터"(성공/초록) · "예시 데이터 표시 중"(주의/호박)
  · "서버 연결 실패"(오류/빨강) · "API 호출 실패 fallback"(주의/호박)이 색만 봐도
  구분되도록 상태별 색을 고정했습니다.
- **모바일 화면**: 시장현황/가격 요약 카드를 2열 그리드로 정리하고, 표 안 여백과
  글자 크기를 조정해 검색순위·최저가·가격순위·[상품 보기] 버튼이 겹치지 않고
  또렷하게 구분되도록 했습니다. 표는 기존처럼 카드 안에서만 가로 스크롤되어
  페이지 전체가 옆으로 밀리지 않습니다.
- **백업**: `backups/index_v3.1.html` 생성.

외부 라이브러리/CDN은 추가하지 않았고, API 키/Secret은 이번에도 `index.html`에
들어가지 않았습니다.

### v3.2 — 헤더 캐릭터 이모티콘 적용 (기능 변경 없음)

**이번 버전도 기능을 추가한 버전이 아니라, v3.1 디자인 위에 상단 헤더의
브랜딩/디자인 요소만 강화한 버전입니다.** 시장현황·상품 검색·상세 결과·사용
안내 기능과 화면 흐름은 v3.1과 동일하게 동작합니다. `server.js`는 이번에도
수정하지 않았습니다.

- **헤더 캐릭터 이모티콘**: `assets/emoticons.png`(2열 x 5행, 캐릭터 10종 스프라이트
  이미지)에서 캐릭터별로 잘라 쓰는 `emoji-happy` / `emoji-drink` / `emoji-thumb` /
  `emoji-hello` / `emoji-shy` / `emoji-heart` / `emoji-think` / `emoji-sleep` /
  `emoji-cry-laugh` / `emoji-tongue` 클래스를 만들고, 그중 6종(happy·thumb·hello·
  heart·think·sleep)을 제목 좌우에, 2종(drink·shy)을 아주 작은 배경 장식으로 헤더에
  배치했습니다. 흰 테두리·부드러운 그림자·살짝 기울어진 회전으로 "떠 있는" 느낌을
  줬습니다.
- **제목은 항상 최상단 레이어**: 제목·부제·기준일을 `.header-content`로 묶어
  이모티콘 레이어(`.header-decor`)보다 위(z-index)에 그리므로, 이모티콘이 절대
  제목 글자를 가리지 않습니다.
- **반응형 개수 조절**: 데스크톱은 8개, 태블릿(900px 이하)은 4개(happy·thumb·
  heart·think), 모바일(700px 이하)은 3개(happy·heart·thumb)만 작게 남기고 나머지는
  `display: none` 처리했습니다. 모바일에서는 제목·부제 영역과 겹치지 않도록 헤더의
  위/아래 여백(패딩) 안에만 배치했습니다.
- **접근성/안정성**: 이모티콘은 장식 요소이므로 `aria-hidden="true"`로 스크린리더가
  건너뛰게 했고, `pointer-events: none`으로 클릭에 영향을 주지 않게 했습니다.
  실제 텍스트("네이버 데이터 참고 대시보드" 등)는 이미지로 대체하지 않고 그대로
  텍스트로 유지했습니다. 이모티콘 배지는 흰 원(배경색)을 기본값으로 깔아두므로,
  이미지가 로드되지 않아도 레이아웃이 깨지지 않고 빈 원만 남습니다.
- **이미지 처리 방식**: `server.js`가 정적 파일 서버 라우트를 갖고 있지 않아
  `assets/emoticons.png`를 URL로 직접 요청하면 404가 나기 때문에(로컬 `node
  server.js` 실행/Render 배포 환경 모두 동일), `server.js`를 건드리지 않고도
  이모티콘이 항상 보이도록 `index.html`의 CSS 안에 `assets/emoticons.png` 원본을
  base64 데이터 URI로 인라인했습니다. 원본 이미지 파일 자체는 `assets/emoticons.png`
  하나만 유지하며(추가로 잘라낸 개별 이미지 파일은 만들지 않음), 이 파일이 곧
  캐릭터 10종의 유일한 원본입니다.
- **백업**: `backups/index_v3.2.html` 생성.

외부 라이브러리/CDN은 추가하지 않았고, API 키/Secret은 이번에도 `index.html`에
들어가지 않았습니다.
